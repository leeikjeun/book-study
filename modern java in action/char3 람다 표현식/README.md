# 람다 표현식


## 람다란?
 - 메서드로 전달할 수 있는 익명 함수를 단순화한 것(책 내용 ㅋ)
 - 인터페이스로 정의되어있는 내용을 함수로 표현하여 처리하는 것

```
 예) // 람다에 사용할 인터페이스 선언 
    Interface LambdaTest{
        String testMethod(Integer value);
    }

    // 인터페이스를 사용하는 메소드 선언
    void testMethod(LambdaTest lambdaTest, Integer value){
        System.out.println(lambdaTest.testMethod(value));
    }
    
    testMethod(
            a -> { // a -> (이부분은 LambdaTest Integer value가 들어온다면을 표현 (파라미터 리스트) 
                    a += 5; // 사실상 value += 5와 같음
                    return Integer.toBinaryString(a); // LambdaTest testMethod 함수의 리턴값인 String을 표현
              }, 10);
    
    // 단순히 한줄로 끝난다면 이와같이 return 및 {}을 없이 한줄로 끝낼 수 있음          
    testMethod(a -> Integer.toBinaryString(a), 10); 
```

## 함수형 인터페이스란?
 - 딱 한개의 메소드만 정의되어있는 인터페이스
    - 한개만 지정해야하는 이유 : 1개 이상인 경우 람다표현식에 어떤 함수가 맵핑되는지 인지를 못함
 - 위의 예제처럼 람다를 사용하고 싶을 때 마다 인터페이스를 만들면 너무 많은 인터페이스가 생김
    - 자바에서 기본적으로 제공해주는 함수형 인터페이스가 있음 의미에 따라 사용하는 것도 좋다고 생각함

### 함수 디스크립터
 - 함수형 인터페이스의 추상 메서드 시그니처는 **람다 표현식의 시그니처** 를 가리킨다.
    - 예)  LambdaTest의 경우 시그니처는 return이 String이고 파라미터가 Integer인 testMethod가 시그니처가됨  
 - 람다 표현식의 시그니처를 서술하는 메서드를 **함수 디스크립터**라

### 람다 실행 : 실행 어라운드 패턴
 - 책 예시 혹은 업무소스 예시 볼 것 ㅎ.
 - 공통된 부분(파일 READ 혹은 디비 SELECT 혹은 트랜잭션 처리 등)을 두고 가운데 동작할 부분을 추상화 하는 형태를 **실행 어라운드 패턴** 이라함

```
   예)
      실행
         추상화된 작업
      정리
``` 

### 함수형 인터페이스 사용
 - java.util.function에 여러가지 형태의 함수형 인터페이스가 있고 상황에 따라 사용해도 좋음
 - 종류는 다양해서 추후 업뎃 예정 ㅎ.
 
#### 기본특화
 - 기본적으로 제공하는 함수형 인터페이스는 재너릭 타입을 사용하고 있으나 기본형(int, double, long..)에 최적화 된 함수형 인터페이스가 있음
 - 필요 이유는 기본형 외 참조형 Integer, Long <-> int, long 간의 변환과정에서 메모리를 좀 더 사용하나 이 변환 동작을 피할 수 있도록 기본특화 인터페이스를 제공하고 있음

#### 지역 변수 사용 (람다 캡처링)
 - 람다에서 지역변수를 익명 함수가 사용하는 것처럼 **자유 변수(파라미터로 넘긴값 외 함수 지역변수를 의미)**를 활용 할 수 있다.
 - 제약조건 : 해당 지역변수를 final 혹은 final처럼 사용해야함

### 메서드 참조
 - 기존 메소드 정의를 재활용해서 람다처럼 전달할 수 있다.
 - 축약하여 사용할 수 있다.
